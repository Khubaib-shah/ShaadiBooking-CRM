DESIGN PHILOSOPHY — NON-NEGOTIABLE
Reference: Skote Admin Template (dark sidebar, clean white content area, 
ApexCharts/Recharts data visualization, professional enterprise feel)

DO NOT build generic "AI dashboard" UI with:
- Generic blue buttons everywhere
- Cookie-cutter card layouts
- Boring default shadcn/ui without customization  
- White backgrounds everywhere with no depth
- Flat, lifeless tables
- Missing hover/focus/active states

DO build:
- Dark sidebar (#2a3042) with subtle active state highlighting
- Content area with slight off-white (#f8f9fa) background
- Cards with gentle box-shadow, not harsh borders
- Micro-animations on interactions (200ms ease-in-out)
- Color-coded status badges with appropriate semantic colors
- Data tables that feel like Notion or Airtable — clean, scannable
- Charts that have tooltips, legends, smooth animations
- Empty states with illustration-style SVG icons, not just text
- Loading skeletons that match the exact shape of loaded content
- Mobile-responsive: sidebar collapses to bottom nav on mobile

SPECIFIC IMPLEMENTATION RULES
typescript// RULE 1: Currency Formatting
// Never use toLocaleString() directly. Use the utility:
// import { formatRs, formatRsCompact } from '@/lib/utils/currency'
// formatRs(770000) === "Rs. 7,70,000"   ← Pakistani lakh formatting
// formatRsCompact(770000) === "Rs. 7.7L"
// formatRs(75000) === "Rs. 75,000"
// formatRsCompact(75000) === "Rs. 75K"

// RULE 2: Pakistani Number Format
// 1,23,456 (not 123,456)
// Lakhs and crores: 1L = 1,00,000  1Cr = 1,00,00,000

// RULE 3: Date Display
// Dates always shown as: "20 Dec 2025" or "20 Dec" (no year if current year)
// Relative: "in 42 days" or "3 days ago"
// Use dayjs consistently, not date-fns mixing

// RULE 4: Phone numbers
// Display as: 0312-1234567
// WhatsApp link: href={`https://wa.me/92${phone.slice(1)}`}

// RULE 5: Loading states
// EVERY query must have: isLoading → skeleton, isError → error state, data → content
// No "undefined" renders — always handle all states

// RULE 6: Optimistic updates
// Marking payment as paid → immediately update UI, rollback on error
// Assigning worker → immediately show in list, rollback on error
// Changing booking status → immediate badge change

// RULE 7: Infinite scroll vs Pagination
// Use Pagination (not infinite scroll) for all data tables
// Reason: Pakistani connection speeds make loading more data slow
// Exception: Activity feeds and notification lists → infinite scroll

// RULE 8: Right-click context menus
// On table rows: right-click → native-feeling context menu
// Options: Open in new tab | Edit | Delete | Copy reference
// Use @radix-ui/react-context-menu

// RULE 9: Keyboard shortcuts
// Cmd/Ctrl + K → global search
// Cmd/Ctrl + N → new booking (from any bookings page)
// Escape → close any open modal/sheet
// Show shortcut hints in tooltips

// RULE 10: Print/PDF generation
// Salary slips, invoices, event deployment sheets
// Use react-to-print for print dialogs
